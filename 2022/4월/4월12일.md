# ✨ 케어닥 기업과제
아직 시작하지는 않았고 시작하기 전에 어떤식으로 할지 생각을 해봤다.  
특히 자료구조를 어떻게 해야 내가 쓰기 편할까? 백엔드가 있어서 API를 사용하면 정말 편할것 같은데...  
진짜 백엔드 공부 해둘걸 하는 생각이 너무 들었다.(다음에 시간나면 간단하게 프로젝트 할 수준까지는 공부해야지)  
우선 list와 Card의 자료를 따로 각자 둘건지 아니면 list객체 안에 배열로 둘지 고민이였다.  
이유는 따로두면 추가할때, Card위치 이동시에 처리하기가 편할 것 같았다.  
하지만 화면에 뿌릴때 filter같은 데이터를 한바퀴 돌리는 과정이 랜더링 할때마다 일어날 것 같아서 그게 싫었다.  
```js
//card
[
  {
    id: 1,
    column: 'backlog',
    title: 'TS공부하기',
    contents: 'TS 5시간 인강보기',
    idx: 1
  },
  {
    id: 2,
    column: 'doing',
    title: 'TIL 작성',
    contents: '오늘 공부한 JS 내용 정리',
    idx: 1
  }
]

// list
[
  {
    id: 1,
    column: 'backlog'
  },
  {
    id: 2,
    column: 'doing'
  },
  {
    id: 3
    column: 'done'
  }
]
```
그래서 아래 처럼 하기로 했다.  
아래처럼하면 title 하나 수정하는데 전체를 다시 바인딩해야한다는게 마음에 안들기는 하지만,
그래도 작은 프로젝트할때 이정도 처리는 그렇게 오류날정도로 오래 걸리지 않으니까 하는 생각에 이렇게 하기로 했다.  
```js
[
  {
    id: 1, 
    column: 'backlog',
    contents: [
        {
          title: 'TIL 작성',
          contents: '오늘 공부한 JS 내용 정리',
        },
        {
          title: 'TS공부하기',
          contents: 'TS 5시간 인강보기',
        }
    ]
  },
  {
    id: 2
    column:'doing',
    contents: []
  },
  {
    id: 3
    column:'done',
    contents: []
  }
]
```
